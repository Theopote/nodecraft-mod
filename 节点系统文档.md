## 一、系统架构

### 1.1 总体流程

- **节点定义**：所有节点位于 `com.nodecraft.nodesystem.nodes` 包下，实现 `INode` 接口（通常继承 `BaseNode` 或 `BaseCustomUINode`）。
- **节点发现**：应用启动时由 `DefaultNodeProvider` 调用 `AutoNodeScanner.scanAndRegisterNodes()`，扫描上述包下的所有实现 `INode` 的类。
- **注册方式**：
  - **注解驱动**：类上标注 `@NodeInfo(id, displayName, description, category)` 时，使用注解信息注册。
  - **约定驱动**：无 `@NodeInfo` 时，从包路径与类名推导分类与显示名（包路径需满足 `nodes.<主分类>.<子分类>`，至少两级）。
- **分类**：主分类由 `DefaultNodeProvider.registerMainCategories()` 注册；子分类在首次注册该分类下的节点时由 `AutoNodeScanner.ensureCategoryExists()` 自动创建。

### 1.2 关键组件

| 组件 | 说明 |
|------|------|
| `NodeRegistry` | 单例注册表，管理所有节点类型（NodeInfo）与分类，支持按 ID 实例化节点。 |
| `INodeProvider` | SPI 接口，实现类在 `META-INF/services/com.nodecraft.nodesystem.spi.INodeProvider` 中声明，用于插件化注册节点。 |
| `DefaultNodeProvider` | 内置实现，注册 9 个主分类并调用 `AutoNodeScanner` 扫描 `com.nodecraft.nodesystem.nodes` 包。 |
| `AutoNodeScanner` | 扫描 `com.nodecraft.nodesystem.nodes` 下所有实现 `INode` 的类，按注解或约定注册到 `NodeRegistry`。 |

## 二、核心 API

### 2.1 节点接口与基类

- **`INode`**：节点契约。提供 `getId()`、`getTypeId()`、`getDisplayName()`、`getDescription()`、`getInputPorts()`、`getOutputPorts()`、`getPositionX/Y()`、`setPosition()`、`compute(inputs)`、`getOutput(portId)`、`setInput(portId, value)`、`getNodeState()`、`setNodeState(state)`。
- **`BaseNode`**：通用节点基类。维护输入/输出端口列表、输入/输出值 Map，实现 `compute()` 中调用 `processNode(ExecutionContext)`；子类实现 `processNode()` 完成业务逻辑。
- **`BaseCustomUINode`**：继承 `BaseNode`，用于需要在编辑器中绘制自定义 UI（如滑动条、颜色选择器、文本框）的节点，提供 ImGui 渲染与尺寸缓存等能力。

### 2.2 端口与数据类型

- **`IPort`**：端口接口。包含 `id`、`displayName`、`description`、`dataType`、`isInput()`、连接状态及 `getValue()`/`setValue()`。
- **`BasePort`**：端口实现类，构造时传入 id、显示名、描述、`NodeDataType`、所属节点。
- **`NodeDataType`**：枚举，定义端口数据类型。常用包括：
  - 基础：`ANY`、`STRING`、`INTEGER`、`DOUBLE`、`BOOLEAN`、`FLOAT`、`COLOR`
  - 几何：`POINT`、`VECTOR`、`COORDINATE`（BlockPos）、`POSITION`、`REGION`、`BOUNDING_BOX`、`LINE`、`POLYLINE`、`CURVE` 等
  - 世界相关：`BLOCK_POS`、`BLOCK_LIST`、`BLOCK_INFO`、`BLOCK_TYPE`、`ENTITY_TYPE`、`ENTITY_INFO`、`ITEM_STACK`、`BIOME`、`NBT_COMPOUND`、`NBT_LIST` 等
  - 列表：`LIST`、`COORDINATE_LIST`、`BLOCK_INFO_LIST`、`VECTOR_LIST`、`REGION_LIST` 等
  - 其他：`FILE_PATH`、`SOUND_EVENT`、`EFFECT_TYPE`、植物相关类型等

### 2.3 节点元数据

- **`@NodeInfo`**（`com.nodecraft.nodesystem.api`）  
  - `id`：节点类型 ID（建议格式 `主分类.子分类.名称`）。  
  - `displayName`：显示名称。  
  - `description`：描述。  
  - `category`：分类 ID，格式 `主分类.子分类`（如 `inputs.basic`、`world.modification`）。  

- **`com.nodecraft.gui.node.NodeInfo`**：注册用 DTO，包含 `id`、`displayName`、`description`、`categoryId`、`nodeClass`，由扫描器或 Provider 构造并交给 `NodeRegistry.registerNode()`。

## 三、节点分类体系

主分类（来自 `DefaultNodeProvider`）与常见子分类对应关系如下（子分类以实际存在节点为准）：

| 主分类 ID | 显示名 | 常见子分类 |
|-----------|--------|------------|
| `animation` | Animation & Effects | transforms, effects, interpolation, time, output |
| `data` | Data & Lists | text, lists, conversion, sequence |
| `flora` | Flora & Nature | generators, modifiers, materials, output |
| `inputs` | Inputs & Parameters | basic, minecraft, selectors, sources |
| `math` | Math & Logic | basic, logic, vector, trigonometry, randomness |
| `spatial` | Spatial & Geometry | voxel, points, generators, arrays, analysis |
| `utilities` | Utilities & Workflow | advanced, fileio, organization |
| `visualization` | Visualization & Execution | preview, debugging, execute |
| `world` | World Interaction | query, modification, nbt, interaction, inventory, entity |
| `control` | （由节点包路径推导） | flow |

## 五、节点开发指南

### 5.1 包与分类约定

- 节点类必须位于 `com.nodecraft.nodesystem.nodes` 下，且包路径至少为 **两层子包**（如 `nodes.inputs.basic`），才能通过约定推导出分类 `inputs.basic`。
- 分类 ID 格式：`主分类.子分类`。主分类需在 `DefaultNodeProvider.registerMainCategories()` 中注册；子分类在首次注册该分类下的节点时自动创建，名称由子包名格式化得到。

### 5.2 使用 @NodeInfo 注册（推荐）

在节点类上添加：

```java
@NodeInfo(
    id = "主分类.子分类.节点_id",
    displayName = "显示名称",
    description = "简短描述",
    category = "主分类.子分类"
)
public class MyNode extends BaseNode { ... }
```

- 若不填 `id`，将使用类名转小写，并视情况加 `_node` 后缀。
- 若不填 `displayName`，将使用类名格式化（如 `MyCustomNode` → `My Custom`）。

### 5.3 实现节点逻辑

1. **继承**  
   - 仅需数据与端口、无自定义 UI：继承 `BaseNode`。  
   - 需在编辑器中绘制自定义控件：继承 `BaseCustomUINode`。

2. **构造函数**  
   - 调用 `super(UUID.randomUUID(), "节点类型ID")`。  
   - 使用 `addInputPort(...)` / `addOutputPort(...)` 添加端口，端口使用 `BasePort` 与 `NodeDataType`。

3. **计算逻辑**  
   - 在 `processNode(ExecutionContext context)` 中：从 `inputValues` 读入，写出到 `outputValues`。  
   - 若有执行阶段（如世界修改），可使用 `ExecutionContext` 获取世界、玩家等上下文。

### 5.4 扩展节点（插件）

1. 实现 `INodeProvider` 的 `registerNodes(NodeRegistry registry)`，在方法内注册分类与节点（可手动构造 `com.nodecraft.gui.node.NodeInfo` 并调用 `registry.registerNode(nodeInfo)`）。
2. 在 JAR 的 `META-INF/services/com.nodecraft.nodesystem.spi.INodeProvider` 中声明实现类全限定名。
3. 将 JAR 放入 classpath，启动时通过 `ServiceLoader` 加载并执行注册。
